# Tom AI Chat Design

## Overview

This document summarizes the proposed design for a Tom AI chat workflow centered on `*.chat.md` files, custom keybindings, and a pair of output channels for tool and response logging.

## Goals

- Provide a dedicated `*.chat.md` file type that users edit directly.
- Allow quick keyboard-driven chat actions (`Start` and `Send`).
- Persist chat state and technical metadata in the chat file.
- Log tool activity separately from final responses.
- Keep newest prompts/responses at the top for quick context.

## File Types and Naming

- **Chat input file:** `<chat-id>.chat.md`
- **Response file:** `<chat-id>.responses.md` (newest response at top)
- **Response summary file:** `<chat-id>.response-summary.md`
- **Location:** All related files live in the same folder as the chat file.
- **Chat ID:** User-created via the `<chat-id>.chat.md` filename.

## Chat File Format

Top of the file includes technical metadata, followed by the active prompt. Older prompts are kept below with separators.

```
toolInvocationToken: <token>
modelId: <model-id>
tokenModelId: <token-model-id>
responsesTokenLimit: <number>
responseSummaryTokenLimit: <number>
contextFilePath: <optional-path-to-context-file>

_________ CHAT <chat-id>____________

... user prompt goes here ...

____________________________________

... old user prompt goes here ...

____________________________________

... older prompts ...
```

### Metadata Fields

| Field | Description | Default |
|-------|-------------|---------|
| `toolInvocationToken` | Token for tool invocations (auto-populated) | Empty |
| `modelId` | Language model ID for chat requests | From settings |
| `tokenModelId` | Model ID for token counting | From settings |
| `responsesTokenLimit` | Max tokens for responses file | 50000 |
| `responseSummaryTokenLimit` | Max tokens for summary file | 8000 |
| `contextFilePath` | Optional path to context file to include | Empty |

### Notes

- Metadata block is always at the top.
- The active prompt sits directly under the `CHAT <chat-id>` header.
- Each completed prompt is separated by a separator line.
- Separator lines are `---` or `___` (at least three characters, may be longer).
- The separator generated by tooling is exactly 30 underscores.
- Separators are fixed (no configuration).

## Commands and Keybindings

- **Command:** `startTomAIChat`
  - **Command ID:** `dartscript.startTomAIChat`
  - **Keybinding:** `Cmd+T`, `Cmd+N`
  - **Scope:** Keybinding applies only to `*.chat.md` files.
  - Creates or initializes a `<chat-id>.chat.md` file.
  - Overwrites related response/summary files when starting a new chat (no format checks).
  - Inserts metadata header and chat title if missing.

- **Command:** `sendToTomAIChat`
  - **Command ID:** `dartscript.sendToTomAIChat`
  - **Keybinding:** `Cmd+T`, `Cmd+S`
  - **Scope:** Keybinding applies only to `*.chat.md` files.
  - Reads the latest prompt block and triggers the model request.
  - Uses a chat participant to obtain a `toolInvocationToken` and execute requested tools.
  - Inserts separators and positions cursor for next prompt after completion.

## Prompt Processing Flow

1. **Pre-flight summary step:**
   - Skip if `<chat-id>.responses.md` does not exist yet.
   - Otherwise generate: `Create a response summary of file <chat-id>.responses.md in file <chat-id>.response-summary.md`.
2. **Prompt assembly template:**
   - Add the summary file as a reference/attachment.
   - Insert the user prompt in a fixed template:

```
This is a new user prompt:

TODO: review the previous responses in attachment <chat-id>.response-summary.md

TODO: the do the following:

<prompt as in the file>.

IMPORTANT: generate intermediate output for console output. Once all tool calls are done, generate final output for the user in markdown format.
```

3. **Tool execution:**
   - Tool call logs go to **Tom AI Chat Log** output channel.
4. **Final response:**
   - Final markdown response goes to **Tom AI Chat Responses** output channel.
   - Stored at top of `<chat-id>.responses.md` with newest first.
  - If `<chat-id>.responses.md` does not exist, create it after the first prompt (even if empty).

## Post-Processing

After a response completes:

- Check size of `<chat-id>.responses.md` and remove oldest entries until size is small enough for input token limit.
- Token estimates use `model.countTokens()` (targeting models like GPT-4o or GPT-4.1).
- Apply a configurable token limit for `<chat-id>.responses.md`.
- Apply a configurable token limit for `<chat-id>.response-summary.md`.
- Insert a separator line and two blank lines **above** the processed prompt block.
- Move the cursor to the new blank prompt area at top, between the two blank lines, ready for input.

## Output Channels

- **Tom AI Chat Log**
  - Tool invocation requests and intermediate output.

- **Tom AI Chat Responses**
  - Final user-facing responses in markdown format.

## Data and Metadata Requirements

- `toolInvocationToken` stored in the chat file header once a chat session is started.
  - Token is sourced from a chat participant request and reused for LM tool calls.
  - Metadata is plain text and user-editable.
- Refresh the chat participant on failure (recreate participant and retry tool call).
- Additional settings may be added in the header block as needed.
- Chat ID is derived from `<chat-id>.chat.md` filename.

## Prompt Extraction Rules

- The active prompt is the text between the `CHAT <chat-id>` header and the first separator line.
- Only the first block is used (single block, may span multiple lines).
- Separator lines are defined as a line starting with `---` or `___` (three or more characters).

## File Update Rules

- On `startTomAIChat`, response and summary files are overwritten (no format checks).
